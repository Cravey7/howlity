# DevFlow

DevFlow is a powerful tool that helps you manage your development workflow. It gives you a simple way to organize systems, applications, features, and tech stacks all in one place.

## What DevFlow Does

DevFlow makes development easier by:

- Organizing your systems and applications clearly
- Tracking features and their development status
- Documenting tech stacks used across your projects
- Showing dependencies between different applications
- Enabling team collaboration on development tasks

## How DevFlow Works

DevFlow provides a comprehensive platform that streamlines your development workflow through:

### System Organization
- **Hierarchical Structure**: Organize your projects in a logical hierarchy of systems, applications, and features
- **Visual Relationships**: See how different components connect and depend on each other
- **Centralized Documentation**: Keep all project information in one accessible location
- **Custom Categorization**: Create your own organizational structure that fits your workflow
- **System Mapping**: Visualize the entire architecture of your development ecosystem

### Application Management

- **Version Control Integration**: Connect with your version control systems for seamless tracking

### Team Collaboration
- **Role-Based Access**: Control who can view and edit different parts of your projects
- **Real-Time Updates**: See changes as they happen across your development ecosystem
- **Comment Threads**: Discuss specific elements directly within the platform
- **Task Assignment**: Assign and track responsibilities for team members

### Reporting and Analytics
- **Custom Dashboards**: Create dashboards that show the metrics most important to you

- **Resource Allocation**: See where your development resources are being utilized
- **Bottleneck Identification**: Identify and address workflow bottlenecks
- **Quality Metrics**: Track code quality and testing coverage
- **Export Capabilities**: Generate reports for stakeholders in various formats

## Getting Started

1. Clone the repository:
   ```bash
   git clone <repository-url>
   cd devflow
   ```

2. Install dependencies:
   ```bash
   npm install
   ```

3. Create a Supabase project and get your project URL and anon key.

4. Create a `.env.local` file in the root directory with the following variables:
   ```
   NEXT_PUBLIC_SUPABASE_URL=your-project-url
   NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
   ```

5. Set up the database schema:
   - Go to your Supabase project's SQL editor
   - Copy the contents of `supabase/schema.sql`
   - Run the SQL commands

6. Start the development server:
   ```bash
   npm run dev
   ```

7. Open [http://localhost:3000](http://localhost:3000) in your browser.
